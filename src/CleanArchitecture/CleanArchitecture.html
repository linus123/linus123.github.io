n-Tier

  • Layers Work = Working Software
  • Each layer depends on the one above.
  • Problem: In the end everything depends on the database.
  • Problem: Dataset tends to penetrate all layers.
  • Problem: Nothing makes you "fall into the pit of success"
  • Problem: Database schemas become VERY hard to change
  • Problem: Hard to changes data sources to services if using dataset
  • Problem: How many of these system have automated test coverage

The Result

  • Tighly coupled to database
  • Now we are legacy
  • Coric Story - Team belives (NOT IT) that once set setup the database that we are going to have a hard time changing it

We can do better.

A Better Architecture

  • Forces abstraction of third party libraries
  • Limits the effect of the database.
  • Defends against legacy code.
  • Supports unit tests.
