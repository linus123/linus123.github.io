Continuous Integration

Team City

Continuous Integration (CI) is a development practice that requires developers to integrate code into a shared repository several times a day. Each check-in is then verified by an automated build, allowing teams to detect problems early.

-- thoughtworks.com
  • Code base make work on each person's machine but may fail when put together.
-- Credit: Aron Cois

Benefits

  • Prevents a "poison" code base.
  • Grantees all code and libraries are in source control.
  • Ensures automated tests are maintained.
  • Can include database.
  • Great place to collection code metrics.
