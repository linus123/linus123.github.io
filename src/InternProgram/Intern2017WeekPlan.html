Service Desk

  • Send an email to IT_ServiceDesk if you have a problem with your computer.

Setup Laptop

  • You need to be granted administrator privileges on your local machine to install software. Please request this by sending an email to the Service Desk.
  • Ensure that "Visual Studio 2015" works on your laptop.
  • Ensure that "SQL Server 2014 Management Studio" works on your laptop.
  • GIT Extensions: Assuming you have administrator privileges on your local machine you can download and install GIT extensions here => https://sourceforge.net/projects/gitextensions/
  • SQL Express: You can install SQL Express by sending an email to the Service Desk and requesting they install the package "DFA - SQL Express SysAdmin Role Config".

Working with GIT

  • Learn to use GIT => Pluralsight: GIT Fundamentals
  • We will create a private key for GIT as a group.
  • Save your private key to your local machine and make a copy to a share drive.
  • Save your PUBLIC key to your account in source control: Dimensional's Private BitBucket

Tiny Returns Project

  • Codebase is located here => Tiny Returns
  • Remember the folder location on your local machine where you choose to clone the codebase.
  • DO NOT clone the codebase to a network drive.
  • Run the local build by running the command 'build.ps1' in Powershell. The build MUST pass!
  • We will run tough a business overview of the Tiny Returns project as a group.

Modify Readme

  • Create a file called 'readme.md' in the root folder of the code base.
  • This file is in markdown format => Markdown Syntax
  • Modify the 'readme.md' file to include all information that is needed to get started with the project.
  • Ensure that all work is going in to a branch with following convention => feature/[You Branch Name]
  • Submit a pull request with your work. We will cover as a group on how to submit pull requests in BitBucket.
  • Your ARE going to get conflicts in GIT / BitBucket. This is by design. We will have to work though them as a team.

Visual Studio with C#

  • Learn to use Visual Studio and an overview of the C# language here => Pluralsight: Visual Studio 2015
  • Demonstrate that you can do the following with Visual Studio using the Tiny Returns project.
    • Run the a project. Any of them will work.
    • Set a break point and have Visual Studio stop at the break point.
    • Show how Step Over, Into, etc..
    • Demonstrate your understanding of the Locals window.
    • Demonstrate your understanding of the Call Stack window.
    • Demonstrate your understanding of the Watch window.
