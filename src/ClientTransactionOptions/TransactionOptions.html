Client Trades

Message Options

Option 1

Client ID Only

Send the client ID any time anything about the client changes.

Pros

  • Dead Simple

Cons

  • Will have to update ENTIRE data set for every message.

Option 2

Client ID and Data Set

Send the client ID and the type of data that has change. Will only send message when the data attached to the client changes.

Pros

  • Still a small message
  • Can update a smaller data set

Cons

  • Will have to identify the set of data sets

Possible Data Set List

  • Account
  • Trade
  • Portfolio?

Option 3

Trade ID Only

Send the trade ID any time anything about the trade changes.

Pros

  • Dead Simple

Cons

Option 4

Trade ID and Action

Send the trade ID any time anything about the trade changes and if the change is an update, insert, or delete.

Pros

  • Still Simple

Cons

  • Changes the type of message to a table like sync.
